## 网络相关

### 计算机网络

>计算机网络体系结构自底向上分为：物理层、数据链路层、网络层、传输层、应用层。

#### **网络层**

>网络层向上只提供简单灵活、无连接、尽最大努力交付的数据报服务。
>
>网络层协议IP是TCP/IP体系中最主要的协议之一，与IP协议配套使用的还有三个协议：
>
>![image-20231127171108936](https://raw.githubusercontent.com/m1crofan/image/main/image-20231127171108936.png)
>
>![image-20231127173907219](https://raw.githubusercontent.com/m1crofan/image/main/image-20231127173907219.png)
>
>- 地址解析协议ARP
>- 网络控制报文协议ICMP
>- 网络组管理协议IGMP
>
>将网络互连药使用一些中间设备。根据中间设备所在的层次，可以有以下四种不同的中间设备：
>
>- 物理层使用的中间设备是**转发器**（repeater）
>- 数据链路层使用的中间设备叫做**网桥**（bridge）
>- 网络层使用的中间设备叫做**路由器**（router）
>- 网络层以下使用的中间设备叫做**网关**（gateway）。
>
>实际上IP地址是标志一台主机（或路由器）和一条链路的**接口**。当一台主机同时连接到两个网络上时，该主机就必须同时具有两个相应的IP地址，其网络号必须是不同的。因此，路由器总是具有两个或两个以上的IP地址。即路由器的每一个接口都有一个不同网络号的IP地址。

##### 地址解析协议ARP

>在实际应用中，已经知道一个机器的IP地址，需要找出其相应的硬件地址。地址解析一些ARP就是用来解决这样的问题。
>
>在网络上可能经常会有新的主机加入进来，或撤走一些主机。**地址解析协议ARP**解决这个问题的方法是在主机ARP高速缓存中存放一个从IP地址到硬件地址的映射表，并且这个映射标还经常动态更新。

##### **IP数据报**

>![image-20231127175307623](https://raw.githubusercontent.com/m1crofan/image/main/image-20231127175307623.png)
>
>**生存时间**：TTL防止无法交付的数据报无限制地在互联网中兜圈子白白消耗网络资源。
>
>**协议**：占8位，协议字段指出此数据报携带的数据是使用何种协议，以便使目的主机的IP层知道应该将数据上交给哪个协议进行处理。

##### **IP地址**

>- 由32位二进制组成，为了便于记忆通常将其转换为点分十进制
>
>```txt
>110110010.01001001.10001100.00000110
>210.73.140.6
>```
>
>- ip地址的形式x.x.x.x x的范围：0-255
>
>- 一个ip地址由两部分组成：网络位+主机位；网段由子网掩码决定
>
>- 根据点分十进制的第一个数字，ISO国际标准组织将ip地址划分为五类
>
>  - A类：1-126默认子网掩码255.0.0.0
>  - B类：128-191默认子网掩码255.255.0.0
>  - C类：192-223默认子网掩码255.255.255.0
>  - D类：224-239组播地址
>  - E类：240-254科研使用
>

#### HTTP

##### 状态码

>- 1XX 表示接收的请求正在处理
>- 2XX表示请求正常处理完毕
>- 3xx需要进行附加操作以完成请求
>- 4xx服务器无法处理请求
>- 5xx服务器处理请求出错
>
>常用的14种状态码
>
>- 200 表示从客户端发来的请求被正常处理
>- 204 代表服务器接收的请求已成功处理，但在返回的响应报文中不含响应头。
>- 206 表示客户端进行了范围请求，响应报文中包含由content-range指定范围的实体内容。
>- 301 永久重定向
>- 302 临时重定向
>- 303 303和302有着同样的功能，但303状态码明确表示客户端应当采用GET方法获取资源
>- 304 表示客户端发送 *附带条件的请求* 时，服务端允许请求访问资源，但未满足条件的情况。304状态码返回时，不包含任何响应体。
>- 401表示授权访问的内容未授权访问。
>- 403
>- 404
>- 500
>- 503
>
>*附带条件的请求*是指采用 GET方法的请求报文中包含 If-Match，If-ModifiedSince，If-None-Match，If-Range，If-Unmodified-Since 中任一首部。

#### 子网掩码

>- 用来确定ip地址的网络位
>- 子网掩码如何确定网络位？（255.0.0.0 ，255.255.0.0，255.255.255.0）
>  - 与255对应的是网络位
>  - 与0对应的是主机位
>- 假设子网掩码为255.255.255.0能用的ip地址为1-254，255为该网段的广播地址
